Programación Orientada
a Objetos


Forma especial de programar, más cercana a la realidad. Se escribe el código en términos de objetos, propiedades y métodos haciendo el código reutilizable.

Objeto

Es una entidad que consta de un estado (atributos) y un comportamiento (métodos) u operaciones que se pueden realizar con él.

Clase

Es la definición de un objeto. Es decir, define sus atributos y sus métodos. Un objeto es una instancia de una clase.

Clase Abstracta

Es una clase que tiene métodos tanto concretos como abstractos (sin implementación). Los métodos abstractos deben ser implementados por las subclases de la clase abstracta. Una clase abstracta no puede ser instanciada y necesita ser extendida para ser usada.

Una clase abstracta proporciona una superclase a partir de la cual otras puedan heredar y compartir un diseño común. Es decir, una clase abstracta define lo que las subclases de ésta SON y aquí se definen los atributos y métodos que les caracterizan, respecto a lo que son.

Ej: La clase abstracta Animal, y el atributo Nombre.

Clase Interfaz

Es como un blueprint de una clase. Puede considerarse como una clase con métodos, pero sin su implementación. Contiene métodos vacíos que representan lo que todas sus subclases deberían tener en común. Las subclases proporcionan la implementación de cada uno de estos métodos. Una clase interfaz es implementada por otra.

Una interfaz describe un conjunto de métodos que se pueden invocar en un objeto. Es decir, una clase interfaz define lo que las subclases de esta PUEDEN realizar, y aquí se establecen los métodos que indican las acciones que quienes la implementen pueden realizar a su manera.

Ej: La clase Pato extiende de la clase abstracta Animal e implementa la interfaz Volar, mientras que la clase Tiburón extiende de igual manera de la clase abstracta Animal pero por su parte implementa la interfaz Nadar.

Clases Abstractas versus Interfaces

Las clases abstractas e interfaces pueden ayudarlo a mantener su diseño más limpio, más organizado y más fácil de mantener.

Las clases abstractas pueden tener constructores y las interfaces no, pero por lo demás son muy similares. Entonces, ¿cuándo debería usar cada uno?

Cuando usa interfaces para componer una clase, la funcionalidad de la clase se amplía mediante las instancias de clase que contiene. La composición tiende a hacer que el código sea más fácil de reutilizar y razonar que la herencia de una clase abstracta. Además, puede usar múltiples interfaces en una clase, pero solo puede crear una subclase de una clase abstracta.

La composición a menudo conduce a una mejor encapsulación, menor acoplamiento (interdependencia), interfaces más limpias y un código más utilizable. Por estas razones, el uso de la composición con interfaces es el diseño preferido. Por otro lado, la herencia de una clase abstracta tiende a encajar de forma natural en algunos problemas. Por lo tanto, debería preferir la composición, pero cuando la herencia tiene sentido, ¡también es permitido hacerlo!



Técnicas de la POO

La programación orientada a objetos está basada en varias técnicas:

Herencia

Permite la creación de clases basadas en clases ya existentes, con las cuales podemos obtener las características de las clases padres, heredando campos, atributos o métodos. Permitiendo la creación de objetos que incorporen atributos y métodos de otros objetos.

Es una relación ES-UN (is-a) entre objetos (extends).

Composición

Composición quiere decir que tenemos una instancia de una clase que contiene instancias de otras clases que implementan las funciones deseadas. Un objeto que tiene como atributo a otro objeto. Es una relación TIENE-UN (has-a) entre objetos (obj in obj).

Polimorfismo

Permite enviar mensajes sintácticamente iguales a objetos de distintos tipos. Es decir, es la capacidad que tienen los objetos para comportarse de múltiples formas. El único requisito que deben cumplir los objetos es que deben saber responder el mensaje que se les envía. Consiste en hacer referencia a objetos de una clase que puedan tomar comportamientos de objetos descendientes de esta.

Polimorfismo dinámico (Override)

O anulación de métodos (Override) ocurre en tiempo de ejecución y es cuando un método es implementado de forma diferente por las clases hijas.

Polimorfismo estático (Overloading)

O sobrecarga de métodos (Overloading) ocurre en tiempo de compilación y es cuando existen dos métodos del mismo nombre pero con diferentes parámetros de entrada en la misma clase.

Cohesión

El nivel de cohesión de una aplicación mide la fuerza de la relación de sus componentes dentro de un módulo. No es una suma de sus componentes sino un conjunto de los mismos. Se refiere a la forma en que podemos agrupar diversas unidades de software para crear una unidad mayor y de esta forma alcanzar un mejor resultado.

Acoplamiento

Es la forma y nivel de interdependencia entre módulos del software. Cuanto más estándar sea la relación de una entidad lógica con otras, mayor re-aprovechamiento se podrá hacer de ella. Si dos unidades de software son completamente independientes la una de la otra, decimos que están desacopladas.

Encapsulamiento

Es el ocultamiento del estado de un objeto, de manera que solo se pueda cambiar mediante las operaciones definidas para ese objeto. Se protegen los datos asociados al objeto contra su modificación por quien no tenga derecho a acceder a ellos. Lo único que se requiere es que el usuario sepa qué se puede hacer y no el cómo se está haciendo.
