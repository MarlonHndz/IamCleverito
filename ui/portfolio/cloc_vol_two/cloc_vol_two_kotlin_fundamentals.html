Fundamentos de Kotlin


En este volumen 2 de CLOC, encontrarás los conceptos necesarios para ser un cleverito en Kotlin. No obstante, he optado por ampliar algunas secciones, ya que, al tratarse de conceptos más extensos, era conveniente ofrecer una definición más detallada. Además, destacar una herramienta que te podrá ser de utilidad como lo es el compilador web de Kotlin .

Declaración de variables

Kotlin utiliza dos palabras clave diferentes para declarar variables: val y var:


> val: Para una variable cuyo valor no cambia nunca.

> var: Para una variable cuyo valor puede cambiar.

> const val: Para crear constantes en tiempo de compilación.


const val difiere de val en que el valor de ésta última se determina en tiempo de ejecución, lo que significa que, por ejemplo, el valor de un val puede ser asignado por una función durante la ejecución del código.

Modificadores de acceso

Los modificadores de acceso en Kotlin son un poco diferentes a Java:

Public:

Por Default todo es Public en Kotlin. Otorga acceso desde cualquier clase y método en cualquier lugar.

Internal:

Permite el acceso únicamente desde el mismo módulo.

Private:

Visibles desde su clase contenedora y sus métodos.

Protected:

Permite el acceso solo a subclases y clases que están dentro del mismo paquete que la clase de miembros protegidos.

lateinit

Palabra clave utilizada para declarar propiedades (variables) que se inicializarán más tarde, pero que no pueden ser nulas. Es aplicable solo para propiedades var y para tipos no primitivos.

Debes inicializar estas propiedades manualmente antes de usarlas, de lo contrario se lanza una excepción (UninitializedPropertyAccessException) si intentas acceder a ellas antes de la inicialización.

Se usa cuando sabes que la variable se inicializará antes de que se acceda, pero no puedes inicializarla en el momento de la declaración (por ejemplo, en onCreate de Android para variables que dependen de un contexto).

lazy

Es una delegación que se utiliza para inicializar una propiedad de forma perezosa (en su primer acceso). Es aplicable únicamente a propiedades val (inmutables), y estas pueden o no ser nulas.

Se usa para inicializar recursos costosos o diferidos de forma automática, como cargar un archivo, establecer conexiones o realizar cálculos pesados. La inicialización ocurre solo la primera vez que se accede a la propiedad.

Es ideal para situaciones en las que no necesitas inicializar un valor inmediatamente, sino solo cuando es realmente necesario.

Strings

Los Strings en Kotlin funcionan de manera muy similar a las cadenas en cualquier otro lenguaje de programación usando " para cadenas y ' para caracteres individuales, y puede concatenar cadenas con el operador +. Puede crear plantillas de String combinandolas con valores; usando $variable se reemplazará con el texto que representa el valor. Esto se llama interpolación de variables.

Además de esto, también es posible insertar una expresión dentro de un String usando ${kotlin expression} con la expresión dentro de las llaves ({}).

Nulabilidad

En kotlin, por defecto una variable no puede ser nula (null) por lo que para poder asignar el valor nulo a una variable hacemos uso del operador de signo de interrogación (?) después del tipo. Por ejemplo:


val number: Int? = null


Generalmente, cuando usamos variables con posible asignación nula debemos validar si esta contiene o no un valor antes de aplicar una operación en la misma; para ello usamos una sentencia if… else sin embargo en kotlin contamos con el Operador Elvis para esta tarea.

Operador Elvis (?:)

El operador Elvis o ?: nos permite realizar una comprobación de seguridad (safe nullability) donde si la expresión de la izquierda al ?: no es nula, el operador la devuelve, de lo contrario, devuelve la expresión de la derecha. Ejemplo:


val a : String? = ”Nullable String value”
val b : Int = a?.length ?: -1


Para este caso tenemos que la variable b no puede ser nula y requerimos una comprobación de seguridad sobre a quien puede o no tener un valor nulo. Para ello usamos el operador Elvis o ?: que nos retorna la 'longitud de a' (a?.length) si 'a' no es nula o -1 en caso de que 'a sea nulo'.

Double Bang (!!)

El operador de aserción no nula o !! (double-bang) nos permite mantener la posibilidad de obtener un NullPointerException cuando nos encontramos con una variable nula. Un ejemplo siguiendo el orden de los anteriores sería:


val c : Int = a!!.length
//Arroja un NullPointerException si 'a' es nulo


Lists, Arrays y Loops

Al igual que en otros lenguajes, en kotlin existen list, arrays y loops que se pueden definir de la siguiente manera:

List:

Para la creación de listas en kotlin tenemos dos opciones:

> Usar listOf para crear una lista que NO puede ser cambiada.

> Usar mutableListOf para una lista cuyos valores pueden cambiar.

Arrays:

Los arrays en kotlin no son mutables; una vez creados, el tamaño de este es fijo. No es posible agregar o remover el tamaño de los elementos, a menos que le copiemos a un nuevo array.

Para declarar un array utilizamos arrayOf y al no tener un tipo de elemento asociado, podemos crear una array que puede llegar a contener diferentes tipos de elementos.

Si deseamos crear un array de un tipo específico, existen constructores para ciertos tipos, por ejemplo intArrayOf() que nos permite crear un array de números enteros.

Loops:

Para la creación de loops podemos utilizar las siguientes declaraciones:

> Usar for para iterar en los elementos de un array o retroceder usando downTo así como el uso de otros operadores.

> Usar while para repetir la siguiente línea una vez cerrada la condición.

> Usar do… while donde repetirá la línea luego de cerrar el while, mientras se cumpla la condición.

> Usar repeat() que repetirá lo indicado dentro, las veces indicadas como parámetro de la función.

Switch vs When

En Java tenemos el uso de switch quien se encarga de comparar la variable que le pasemos por parámetro con varias declaraciones case presentes correspondientes a ella y realiza la operación pertinente.

Por otro lado, en kotlin tenemos when que compara la variable con varias opciones disponibles; si encuentra más de una coincidencia solo se considerará la que se produzca primero y en caso de no encontrar ninguna se ejecutará el else.



when(number) {
1 -> println("One")- - - - - - - - - - - - - - - //statement 1
2,3 -> println("Two or Three")- - - - - - - - - -//statement 2
in 4..8 -> println("Number between 4 and 8")- - -//statement 3
else -> println("Number is not between 1 and 8") //statement 4
}


when {
number < 1 -> println("Number is less than 1")- -//statement 5
number > 1 -> println("Number is greater than 1")//statement 6
}


when(x) {
is Int -> println("X is integer")- - - - - - - - //statement 7
is String -> println("X is string")- - - - - - - //statement 8
}


Además, podemos ver varias de las ventajas de when permitiendo:

> Verificar o comparar más de una condición que llevan al mismo resultado (statement 2).

> Uso de condiciones o rama de condición, donde el cuerpo de la declaración se ejecutará cuando se cumpla la condición (statement 3).

> when sin argumento, que nos permite utilizar el parámetro de la función directamente en las declaraciones condicionales (statements 5 y 6).

> Transmisión inteligente, que permite una transmisión automática si está usando una expresión is en el bloque condicional (statements 7 y 8).
